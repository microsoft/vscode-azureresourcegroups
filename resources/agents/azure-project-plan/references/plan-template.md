## 11. Build Phases

> This section is a high-level phase summary. Use it as an outline; do NOT mutate it during execution. There is no separate checklist file тАФ progress is reflected by the plan's `Status:` field (`Planning тЖТ Approved тЖТ In Progress тЖТ Awaiting UX Approval тЖТ Awaiting Integration тЖТ Integrating тЖТ Integrated`).
>
> Each phase has a test gate (ЁЯзк). The agent MUST run tests and verify they pass before proceeding. If tests fail, iterate on the code until green.

### Phase 1: Planning
- [ ] Analyze workspace (mode: NEW / AUGMENT)
- [ ] Gather requirements (runtime, services, frontend, features)
- [ ] Select test runner
- [ ] Select Azure services
- [ ] Design project structure
- [ ] Define routes
- [ ] Define test suite plan
- [ ] Write `.azure/project-plan.md`
- [ ] Present plan тАФ get user approval

### Phase 2: Execution

#### Step 1: Foundation
- [ ] Initialize project config ({package.json / .csproj})
- [ ] Configure TypeScript / JavaScript / .NET build
- [ ] Configure linter and formatter
- [ ] Configure test runner ({vitest / jest / mocha+chai+sinon / xUnit / NUnit})
- [ ] Create directory structure
- [ ] Create `.gitignore`
- [ ] ЁЯзк **Test Gate**: Project builds with zero errors; test runner executes cleanly

#### Step 2: Configuration & Environment
- [ ] Create config module with env var loading
- [ ] Create `.env.example` with all variables
- [ ] Create `local.settings.json` with emulator defaults
- [ ] Implement startup env validation (fail fast on missing required vars)
- [ ] Write config unit tests (load, defaults, missing var error)
